# Phalanx-OS 시스템 확장 로드맵 (MECE)

> **기준일:** 2026-05-01 | **현재 완성 스택:** P1~P5 + 개인 미디어 허브(v1)
> 전체 확장 방향을 **6개 독립 도메인**으로 분류. 각 도메인은 상호 배타적이며 합산 시 시스템 전체를 포괄합니다.

---

## 도메인 구조 (6-Domain MECE)

```
┌─────────────────────────────────────────────────────────┐
│  1. 인증·권한     │  2. 미디어 배포   │  3. 여론 인텔리전스  │
├─────────────────────────────────────────────────────────┤
│  4. 참여자 생태계 │  5. 정치인 센터   │  6. 시스템 인프라   │
└─────────────────────────────────────────────────────────┘
```

---

## 1. 인증·권한 (Auth & RBAC)
> **범위:** 사용자 식별, 권한 제어, 접근 관리

### 1-A. 실제 Auth 서비스 연동 (현재: localStorage mock)
| 항목 | 내용 | 우선순위 |
|:-----|:-----|:--------:|
| Supabase Auth 연동 | 이메일/소셜 로그인 → 실제 `auth.uid()` | ⭐⭐⭐ |
| 카카오 OAuth | 한국 사용자 주요 로그인 채널 | ⭐⭐⭐ |
| 구글 OAuth | 2차 소셜 로그인 | ⭐⭐ |
| 초대 전용 가입 | PRINCIPAL/EXPERT는 관리자 초대 코드로만 가입 | ⭐⭐⭐ |

### 1-B. 역할 고도화
| 항목 | 내용 | 우선순위 |
|:-----|:-----|:--------:|
| 역할 신청·승인 워크플로 | CITIZEN → ACTIVIST 전환 신청 + STRATEGIST 승인 | ⭐⭐⭐ |
| 지역 기반 권한 (REGION_LEAD) | 소속 지역 태그로 참여자 범위 제한 | ⭐⭐ |
| 임시 권한 위임 | PRINCIPAL이 특정 EXPERT에게 기간 한정 승인권 부여 | ⭐ |
| MFA 강제 적용 | PRINCIPAL/SYSOP 계정에 TOTP 필수화 | ⭐⭐ |

### 1-C. 감사 추적 (Audit Trail)
| 항목 | 내용 |
|:-----|:-----|
| 역할 변경 이력 로그 | 누가 언제 어떤 역할로 변경되었는지 |
| 승인 행위 로그 | PRINCIPAL의 팩트카드 승인/반려 이력 |
| 로그인 이상 탐지 | 비정상 접근 알림 (Telegram webhook) |

## 2. 개인 미디어 허브 Phase 2~3
> **범위:** 콘텐츠 생성 → 배포 → 추적 → 분석 전 주기

### 2-A. 배포 자동화 (현재: 링크 복사 only)
| 항목 | 내용 | 우선순위 |
|:-----|:-----|:--------:|
| X(트위터) API 직접 게시 | OAuth 2.0 → 버튼 한 번으로 X 게시 | ⭐⭐⭐ |
| 카카오 공유 SDK | KakaoShare API → 채팅방 바로 전송 | ⭐⭐⭐ |
| 텔레그램 Bot API | 지정 채널/그룹에 자동 포스팅 | ⭐⭐ |
| 네이버 블로그 API | 초안 자동 업로드 (오픈API) | ⭐ |
| 인스타그램 Graph API | Meta 비즈니스 계정 연동 | ⭐ |

### 2-B. 콘텐츠 스케줄링
| 항목 | 내용 |
|:-----|:-----|
| 예약 발송 | 날짜/시간 지정 → Supabase pg_cron 실행 |
| 반복 배포 | 주요 입장 자동 주기 재배포 (캠페인 기간) |
| 최적 시간대 추천 | 과거 클릭 패턴 기반 AI 추천 |

### 2-C. 성과 분석 대시보드
| 항목 | 내용 |
|:-----|:-----|
| 채널별 클릭·공유 비교 | 어느 채널이 가장 효과적인지 |
| 콘텐츠별 전환율 | 공론장 유입 → 실제 참여 전환 |
| UTM 코호트 분석 | 캠페인 기간별 누적 성과 |
| REGION_LEAD 팀 집계 | 조직 내 참여자들의 배포 성과 합산 |

### 2-D. AI 콘텐츠 어시스턴트 (현재: 수동 작성)
| 항목 | 내용 |
|:-----|:-----|
| 채널별 자동 포맷 변환 | 공식 입장 → X용(280자)/카카오용/블로그용 자동 변환 |
| 톤 조절 슬라이더 | 공식체 ↔ 친근체 선택 |
| 해시태그 자동 생성 | 여론 키워드 기반 최적 해시태그 추천 |
| 이미지 카드 자동 생성 | 텍스트 → OG 이미지 자동 제작 (Vercel OG) |

## 6. 시스템 인프라·운영
> **범위:** 보안, 성능, 운영 자동화, 모니터링

### 6-A. 보안 강화
| 항목 | 내용 | 우선순위 |
|:-----|:-----|:--------:|
| Supabase RLS 단위 테스트 | pgTAP 기반 정책 자동화 검증 | ⭐⭐⭐ |
| OWASP 점검 | XSS/CSRF/SQLi 취약점 스캔 | ⭐⭐ |
| 비밀 키 로테이션 자동화 | Vercel 환경변수 주기적 갱신 | ⭐⭐ |
| DDoS 방어 | Vercel Edge Rate Limiting 고도화 | ⭐⭐ |

### 6-B. 성능·확장성
| 항목 | 내용 |
|:-----|:-----|
| Supabase Edge Functions | 무거운 AI 처리 → 서버리스 엣지로 분리 |
| 이미지 최적화 CDN | 소셜 카드 이미지 Cloudflare R2 캐싱 |
| DB 인덱스 최적화 | 여론 쿼리 실행 계획 분석 |
| Incremental Static Regeneration | 공론장 페이지 ISR 적용 |

### 6-C. 운영 자동화
| 항목 | 내용 |
|:-----|:-----|
| 일일 여론 리포트 | Cron → PDF 생성 → 텔레그램/이메일 발송 |
| 이상 알림 | 클릭 수 급감, 여론 급변 시 즉시 알림 |
| 자동 백업 | Supabase 일별 스냅샷 → Cloudflare R2 |
| Vercel 배포 상태 모니터 | 빌드 실패 → Slack/텔레그램 알림 |

### 6-D. 데이터 거버넌스
| 항목 | 내용 |
|:-----|:-----|
| 개인정보 익명화 파이프라인 | 30일 후 IP 해시 자동 삭제 |
| 데이터 보존 정책 | 원시 여론 데이터 90일 후 집계 후 삭제 |

## 우선순위 실행 매트릭스

```
                     높은 영향력
                          │
    Supabase Auth ────────┤──── X API 직접 게시
    역할 신청·승인 ────────┤──── AI 일일 브리핑
    의제 버스트 감지 ──────┤──── 카카오 OAuth
                          │
낮은 ──────────────────────┼─────────────────── 높은 구현 난이도
 구현                      │
  난이도                   │
    지지율 트래커 ──────────┤──── 유튜브 댓글 수집
    공개 설문 참여 ─────────┤──── 시계열 예측 모델
    기여 배지 ──────────────┤──── 허위정보 탐지
                          │
                     낮은 영향력
```

## 단계별 실행 계획

| Phase | 기간 | 핵심 과제 |
|:------|:-----|:---------|
| **Phase 2** | 2~4주 | Supabase Auth 연동, X/카카오 API 배포 자동화, 스케줄링 |
| **Phase 3** | 1~2개월 | 여론 채널 확대, AI 보좌관 RAG, 채널별 콘텐츠 변환 |
| **Phase 4** | 2~3개월 | 지지율 트래커, 예측 모델, 전문가 프로필 생태계 |
| **Phase 5** | 3개월+ | 의정 연동, 오프라인 행사 관리, 데이터 거버넌스 완성 |

> [!NOTE]
> 모든 Phase는 **현재 Vercel + Supabase 스택을 유지**하며 점진적으로 확장됩니다.
> 소셜 API 연동(X, Kakao)은 각 플랫폼의 개발자 계정 신청이 선행되어야 합니다.

> [!TIP]
> **가장 빠른 ROI:** Supabase Auth 실제 연동 → 역할 신청·승인 워크플로 → X API 직접 게시 순서로 진행하면 실제 운용 가능한 시스템으로 전환됩니다.
